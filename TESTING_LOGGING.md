# Testing Parser Logging

Полное руководство по тестированию логирования парсера.

## Быстрый старт

### Запуск парсера standalone (без бота)

```bash
# Базовый запуск с INFO логами (полная воронка парсинга)
python run_parser_standalone.py

# С DEBUG логами (полный анализ полей)
LOG_LEVEL=DEBUG python run_parser_standalone.py

# С видимым браузером
HEADLESS=false python run_parser_standalone.py
```

## Интерпретация результатов

### Хороший результат
- ✅ Большой процент объектов на этапе [3/4] (означает хорошую фильтрацию)
- ✅ Для каждого объекта видны все ключевые поля
- ✅ [ОБЪЕКТ] карточка содержит название, категорию, регион
- ✅ ТЭП отмечены как "есть" для большинства объектов

### Проблемы
- ⚠️ Если на [3/4] видно слишком много объектов (>50% от [1/4]) — может быть неправильный фильтр по дате
- ⚠️ Если на карточке объекта видны одни прочерки (—) — нет данных на странице
- ⚠️ Если нет [4/4] сообщения — ошибка при отправке уведомления

## Тестовые сценарии

### 1. Проверить логирование фильтров
```bash
# Установить временный фильтр на одну категорию
# В админ-панели или напрямую в БД изменить filter_categories
LOG_LEVEL=INFO python run_parser_standalone.py
# Убедиться что в блоке "ПАРАМЕТРЫ ЗАПУСКА" видна 1 категория
```

### 2. Проверить парсинг полей
```bash
LOG_LEVEL=DEBUG python run_parser_standalone.py
# Проверить что для каждой карточки видны найденные и пустые поля
```

### 3. Проверить двухфазный парсинг
```bash
LOG_LEVEL=DEBUG python run_parser_standalone.py
# Проверить в логах соотношение Фазы 1 (ID-based) и Фазы 2 (label-pairs)
```

### 4. Проверить фильтр по дате
```bash
# Установить последний запуск на сегодня
LOG_LEVEL=DEBUG python run_parser_standalone.py
# Проверить что видны логи ОТСЕЯН/ОСТАВЛЕН для каждого объекта
```

## Расположение логов

Логи сохраняются в:
```
logs/parser-YYYY-MM-DD.log
```

Просмотр в реальном времени:
```bash
tail -f logs/parser-$(date +%Y-%m-%d).log
```

## Примечания

- Логирование синхронизировано с фактическим потоком выполнения
- INFO уровень показывает полную воронку без излишнего шума
- DEBUG уровень содержит информацию для диагностики проблем парсинга
- Все временные метки в UTC (как определено в `last_successful_run`)
