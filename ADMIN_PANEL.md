# Админ-панель парсера Vitrina

## Быстрый старт

### 1. Добавить администраторов в .env

Откройте файл `.env` и добавьте свой Telegram ID:

```env
ADMIN_ID=123456789
```

**Как узнать свой Telegram ID:**
1. Напишите боту [@userinfobot](https://t.me/userinfobot)
2. Он отправит ваш ID (например, `123456789`)

Можно добавить несколько ID через запятую:
```env
ADMIN_ID=123456789,987654321,555555555
```

### 2. Перезапустить парсер

После изменения `.env` перезапустите парсер:
```bash
python -m src.main
```

### 3. Открыть админ-панель

Отправьте боту команду:
```
/admin
```

Или используйте кнопку в главном меню бота.

### ⏰ Расписание (cron)

Формат: `минута час день месяц день_недели`

**Готовые пресеты:**
```
0 6 * * *      # Ежедневно в 6:00 UTC
0 9 * * *      # Ежедневно в 9:00 UTC
0 12 * * *     # Ежедневно в 12:00 UTC
0 9 * * 1-5    # В 9:00 по будням (Пн-Пт)
0 6 * * 1-5    # В 6:00 по будням (Пн-Пт)
```

**Настройка по параметрам:**
- 🕐 **Выбрать час** — выбор часа запуска (0-23)
- 📅 **Дни недели** — ежедневно / будни / выходные
- 🔄 **Интервал** — запуск каждые N часов (2, 3, 4, 6, 8, 12)
- ✏️ **Ввести cron вручную** — произвольное cron-выражение

**Примеры cron:**
```
0 6 * * *      # Ежедневно в 6:00 UTC
0 9 * * 1-5    # В 9:00 по будням
0 */4 * * *    # Каждые 4 часа
0 0 * * 0      # Каждое воскресенье в полночь
30 7 * * *     # Ежедневно в 7:30 UTC
```

**Как изменить:**
1. Нажмите "⏰ Расписание"
2. Выберите готовый пресет или настройте вручную
3. Изменения применяются немедленно

### 📱 Чаты для уведомлений

Управление чатами для отправки уведомлений о новых проектах.

**Типы чатов:**
- **Приватные** — личный чат с ботом (ID: положительное число)
- **Групповые** — группы и каналы (ID: отрицательное число)

**Как добавить чат:**
1. Нажмите "📱 Чаты" в админ-панели
2. Отправьте `/getChatId` в нужный чат
3. Бот ответит с ID чата
4. Отправьте ID сообщением в админ-панель

**Управление чатами:**
- 🔔 **Включить** — активировать чат для уведомлений
- 🔕 **Выключить** — временно отключить чат
- 🗑 **Удалить** — удалить чат из списка

**Важно:**
- Уведомления отправляются во все активные чаты
- Если чатов нет, используется основной чат из `.env`
- Бот должен быть участником группы/канала

**Подробнее:** [CHATS_MANAGEMENT.md](./CHATS_MANAGEMENT.md)

## Архитектура

### Файлы

| Файл | Описание |
|------|----------|
| `src/services/admin_panel.py` | Сервис админ-панели (FSM, callback handlers) |

### Таблицы БД

**parser_settings:**
```sql
CREATE TABLE parser_settings (
    id              INTEGER PRIMARY KEY,
    key             TEXT UNIQUE,
    value           TEXT,
    description     TEXT,
    updated_at      TEXT
)
```

**admins:**
```sql
CREATE TABLE admins (
    id              INTEGER PRIMARY KEY,
    telegram_id     TEXT UNIQUE,
    username        TEXT,
    created_at      TEXT
)
```

**notification_chats:**
```sql
CREATE TABLE notification_chats (
    id              INTEGER PRIMARY KEY,
    chat_id         TEXT UNIQUE,
    chat_name       TEXT,
    is_active       INTEGER DEFAULT 1,
    created_at      TEXT
)
```

**credentials:**
```sql
CREATE TABLE credentials (
    id              INTEGER PRIMARY KEY,
    login           TEXT,
    password        TEXT,
    label           TEXT,
    is_active       INTEGER DEFAULT 0,
    created_at      TEXT
)
```

### Хранение настроек

Настройки сохраняются в БД в виде JSON:

| Ключ | Формат | Пример |
|------|--------|--------|
| `filter_categories` | JSON array | `["Жилые объекты..."]` |
| `filter_regions` | JSON array | `["Москва", "Тюмень"]` |
| `expertise_year` | Integer | `2025` |
| `cron_schedule` | String | `0 6 * * *` |

## Безопасность

- Только администраторы (из `ADMIN_ID` или БД) могут использовать `/admin`
- Администраторы могут добавлять других через `/add_admin`
- Настройки хранятся в БД и сохраняются между перезапусками
- Права проверяются по комбинации: `.env` + БД
- Учетные данные шифруются в БД

## Синхронизация админов

При каждом запуске парсера:
1. Считываются ID из `ADMIN_ID` в `.env`
2. Добавляются в БД (таблица `admins`)
3. Права проверяются по обоим источникам

**Важно:** Изменения в `.env` требуют перезапуска парсера!

## Troubleshooting

### ❌ "У вас нет прав администратора"

Вы не добавлены в таблицу admins. Обратитесь к текущему администратору.

**Решение:**
- Добавьте свой ID в `ADMIN_ID` в `.env`
- Перезапустите парсер
- Или попросите администратора добавить вас через `/add_admin`

### ❌ Кнопки не работают

Проверьте логи:
```bash
tail -f logs/parser-*.log
```

### ❌ Настройки не сохраняются

Проверьте БД:
```bash
sqlite3 data/vitrina.db "SELECT * FROM parser_settings;"
```

### ❌ Уведомления не приходят в чат

**Проверьте:**
1. Бот добавлен в чат?
2. Чат активен (не выключен в настройках)?
3. ID чата правильный?

**Решение:**
- Удалите и заново добавьте чат
- Проверьте права бота в группе

### ❌ Ошибка авторизации на портале

**Проверьте:**
1. Учетка активна?
2. Логин/пароль правильные?

**Решение:**
- `/admin` → "🔑 Учетки"
- Добавьте новую учетку
- Выберите её активной

## API для разработчиков

### Получить настройки из кода

```python
from src.db.repository import Repository
from src.db.database import Database

repo = Repository(Database("data/vitrina.db"))
settings = repo.get_all_settings()

print(settings.filter_categories)
print(settings.filter_regions)
print(settings.expertise_year)
print(settings.cron_schedule)
```

### Сохранить настройки

```python
from src.db.repository import ParserSettings

settings = ParserSettings(
    filter_categories=["Жилые объекты"],
    filter_regions=["Москва"],
    expertise_year=2025,
    cron_schedule="0 6 * * *"
)

repo.save_settings(settings)
```

### Проверить права администратора

```python
if repo.is_admin("123456789"):
    print("Администратор")
```

### Управление чатами

```python
# Добавить чат
repo.add_notification_chat("-1001234567890", "Мой канал")

# Получить активные чаты
chats = repo.get_notification_chats()

# Выключить чат
repo.toggle_notification_chat("-1001234567890")

# Удалить чат
repo.remove_notification_chat("-1001234567890")
```

### Управление учетками

```python
# Добавить учетку
repo.add_credential("login@example.com", "password", "Основная")

# Получить активную
active = repo.get_active_credential()

# Переключить учетку
repo.set_active_credential(cred_id)

# Получить все
creds = repo.get_all_credentials()
```
