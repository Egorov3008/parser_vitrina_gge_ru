# Управление чатами для уведомлений

## Обзор

Функционал позволяет отправлять уведомления о новых проектах в несколько чатов одновременно. Администраторы могут управлять списком чатов через админ-панель или команды бота.

**Возможности:**
- Добавление неограниченного количества чатов
- Поддержка личных, групповых чатов и каналов
- Включение/выключение чатов без удаления
- Автоматическая отправка во все активные чаты

---

## Быстрый старт

### 1. Получить ID чата

Отправьте боту команду:
```
/getChatId
```

Бот ответит сообщением с ID чата:
```
📱 ID вашего чата:

123456789

Тип: private
```

### 2. Добавить чат через админ-панель

1. Откройте админ-панель: `/admin`
2. Нажмите **📱 Чаты**
3. Отправьте ID чата сообщением
4. Готово! ✅

### 3. Проверить список

В админ-панели отображаются все добавленные чаты:
```
📱 Чаты для отправки уведомлений

✅ Личный чат (123456789)
✅ Рабочий канал (-1001987654321)
🔕 Тестовый чат (-1001234567890)
```

---

## Типы чатов

### Личные чаты (private)

**ID:** положительное число (например, `123456789`)

**Как добавить:**
1. Отправьте `/getChatId` в личном чате с ботом
2. Скопируйте ID
3. Добавьте через админ-панель

**Особенности:**
- Бот общается с вами напрямую
- Не требует добавления бота в какие-либо группы
- ID всегда положительный

---

### Групповые чаты (group/supergroup)

**ID:** отрицательное число (например, `-1001234567890`)

**Как добавить:**
1. Добавьте бота в группу
2. Отправьте `/getChatId` в группе
3. Бот ответит в личном чате с ID группы
4. Скопируйте ID и добавьте через админ-панель

**Особенности:**
- Бот должен быть участником группы
- ID всегда отрицательный
- Для каналов ID начинается с `-100`

**Права бота в группе:**
- Отправка сообщений ✅
- Чтение сообщений (не обязательно)

---

### Каналы (channel)

**ID:** отрицательное число, начинающееся с `-100` (например, `-1001987654321`)

**Как добавить:**
1. Добавьте бота в канал как администратора
2. Отправьте `/getChatId` в канале (или перешлите сообщение из канала боту)
3. Скопируйте ID
4. Добавьте через админ-панель

**Особенности:**
- Требуется роль администратора для бота
- ID всегда начинается с `-100`
- Бот может только отправлять сообщения

---

## Управление чатами

### Добавление чата

#### Способ 1: Через админ-панель (рекомендуется)

1. `/admin` → **📱 Чаты**
2. Отправьте ID чата сообщением
3. Чат появится в списке

#### Способ 2: Автоматически при первом сообщении

Если вы отправляете команду боту в новом чате, он автоматически запоминает этот чат.

---

### Включение/выключение чата

Без удаления можно временно отключить чат:

1. `/admin` → **📱 Чаты**
2. Нажмите **🔔/🔕** рядом с чатом
3. Статус изменится:
   - ✅ **Вкл** — чат активен
   - ❌ **Выкл** — чат отключен

**Зачем:**
- Временная пауза уведомлений
- Тестирование настроек
- Сезонное отключение

---

### Удаление чата

1. `/admin` → **📱 Чаты**
2. Нажмите **🗑** рядом с чатом
3. Подтвердите удаление

**Важно:** Удаление необратимо!

---

## Поведение системы

### Если нет добавленных чатов

- Уведомления отправляются в **основной чат** из `TELEGRAM_CHAT_ID` в `.env`
- Это поведение по умолчанию для обратной совместимости

### Если добавлены чаты в админ-панели

- Уведомления отправляются во **все активные чаты**
- Основной чат из конфига игнорируется
- При ошибке отправки в один чат, остальные получают уведомление

### При ошибке отправки

- Бот продолжает отправку в остальные чаты
- Ошибка логируется в файл логов
- Парсер не прерывается
- Вы получаете уведомление об ошибке в лог

## Примеры использования

### Пример 1: Уведомления в рабочий канал

**Задача:** Отправлять уведомления в канал отдела инвестиций

**Шаги:**
1. Добавить бота в канал как администратора
2. Отправить `/getChatId` в канале
3. Бот ответит: `ID: -1001987654321`
4. `/admin` → **📱 Чаты**
5. Отправить ID: `-1001987654321`
6. ✅ Готово

---

### Пример 2: Несколько получателей

**Задача:** Уведомления должны приходить:
- Лично вам
- В рабочую группу
- В канал руководителя

**Решение:**
1. Добавить личный чат: `123456789`
2. Добавить рабочую группу: `-1001234567890`
3. Добавить канал: `-1001987654321`

Теперь каждое уведомление отправляется во все 3 чата одновременно.

---

### Пример 3: Временное отключение

**Задача:** На выходные отключить уведомления в рабочую группу

**Решение:**
1. `/admin` → **📱 Чаты**
2. Нажать **🔕** рядом с рабочей группой
3. В понедельник нажать **🔔** для включения

---

### Пример 4: Замена канала

**Задача:** Старый канал закрыт, нужно перейти на новый

**Решение:**
1. Добавить новый канал: `-1001111111111`
2. Удалить старый канал: `-1001234567890`
3. Проверить, что уведомления идут в новый канал

## Технические детали

### База данных

Таблица `notification_chats`:

```sql
CREATE TABLE notification_chats (
    id          INTEGER PRIMARY KEY,
    chat_id     TEXT UNIQUE NOT NULL,
    chat_name   TEXT,                 -- Опциональное название
    is_active   INTEGER DEFAULT 1,    -- Флаг активности
    created_at  TEXT DEFAULT (now)    -- Время добавления
);
```

### API методы

**Repository:**

```python
# Добавить чат
repo.add_notification_chat(chat_id: str, chat_name: Optional[str] = None)

# Получить активные чаты
chats = repo.get_notification_chats()  # Только активные

# Получить все чаты
all_chats = repo.get_all_notification_chats()  # Все

# Переключить активность
new_state = repo.toggle_notification_chat(chat_id: str)

# Удалить чат
repo.remove_notification_chat(chat_id: str)
```

**TelegramService:**

```python
# Отправить уведомление
await telegram.send_notification(
    message: str,
    chat_ids: Optional[List[str]] = None
)

# Отправить alert об ошибке
await telegram.send_alert(
    error_message: str,
    chat_ids: Optional[List[str]] = None
)
```

## Решение проблем

### Бот не отправляет уведомления в групповой чат

**Проверьте:**
1. Бот добавлен в групповой чат?
2. У бота есть права на отправку сообщений?
3. Чат активен (не выключен в настройках)?
4. ID чата правильный (должен быть отрицательным)?

**Решение:**
```bash
# Проверить в БД
sqlite3 data/vitrina.db "SELECT * FROM notification_chats;"

# Удалить и добавить заново
# /admin → Чаты → Удалить → Добавить новый ID
```

---

### Ошибка "Chat not found"

**Причины:**
- ID неправильный
- Бот был удален из чата/канала
- Чат был заблокирован или удален
- Бот не администратор в канале

**Решение:**
1. Удалить этот чат из админ-панели
2. Проверить, что бот все еще участник чата
3. Для канала: проверить права администратора
4. Добавить чат заново

---

### Уведомления идут только в один чат

**Вероятная причина:**
- В админ-панели не добавлены другие чаты
- Остальные чаты выключены

**Решение:**
1. `/admin` → **📱 Чаты**
2. Проверить список чатов
3. Включить нужные чаты (🔔)
4. Добавить новые при необходимости

---

### Команда `/getChatId` не работает в группе

**Причины:**
- Бот не добавлен в группу
- У бота нет прав на чтение сообщений

**Решение:**
1. Добавить бота в группу
2. Дать права на чтение сообщений
3. Отправить `/getChatId` снова

---

### Ошибка отправки в конкретный чат

**Логирование:**
```bash
# Проверить логи на ошибки
grep "Error sending" logs/parser-*.log

# Посмотреть детали
tail -f logs/parser-$(date +%Y-%m-%d).log
```

**Решение:**
1. Проверить ID чата в логах
2. Удалить проблемный чат
3. Добавить заново

## Рекомендации

### Безопасность
- ❌ Не публикуйте ID чатов публично
- ✅ Используйте приватные ссылки для групп
- ✅ Проверяйте права доступа бота

### Производительность
- ✅ Отправка в 3-5 чатов не влияет на скорость
- ⚠️ При 10+ чатах возможна задержка 1-2 секунды
- ✅ Используйте активные чаты (выключенные не проверяются)

### Мониторинг
- ✅ Регулярно проверяйте логи: `tail -f logs/parser-*.log`
- ✅ Тестируйте новые чаты перед продуктивным использованием
- ✅ Настройте уведомления об ошибках

### Резервное копирование
- ✅ Основной чат из `.env` работает как fallback
- ✅ Храните список важных чатов отдельно
- ✅ Экспортируйте настройки периодически

## Сценарии использования

### Сценарий 1: Личный мониторинг

**Конфигурация:**
- 1 личный чат

**Плюсы:**
- Простота настройки
- Никто не видит уведомления
- Полный контроль

---

### Сценарий 2: Командная работа

**Конфигурация:**
- 1 рабочая группа
- 1 личный чат (для админа)

**Плюсы:**
- Вся команда в курсе новых проектов
- Админ получает уведомления лично
- Можно обсуждать проекты в группе

---

### Сценарий 3: Мульти-проект

**Конфигурация:**
- Группа проекта А
- Группа проекта Б
- Канал руководства
- Личный чат админа

**Плюсы:**
- Разделение по проектам
- Руководство в отдельном канале
- Централизованный контроль

---

### Сценарий 4: Тестирование и продакшен

**Конфигурация:**
- Тестовый чат (выключен)
- Продакшен чат (включен)

**Плюсы:**
- Быстрое переключение между средами
- Тесты не беспокоят продакшен
- Легкое развертывание
