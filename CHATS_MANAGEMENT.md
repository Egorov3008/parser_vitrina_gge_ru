# Управление чатами для уведомлений

## Обзор

Новый функционал позволяет отправлять уведомления о новых проектах в несколько чатов одновременно. Администраторы могут управлять списком чатов через админ-панель.

## Как использовать

### 1. Получение ID чата

**Команда: `/getChatId`**

Отправьте эту команду в любом чате, и бот ответит с ID этого чата.

Например, в приватном чате бот ответит:
```
📱 ID вашего чата:

123456789

Тип: private
```

В групповом чате (если бот добавлен как участник):
```
📱 ID вашего чата:

-1001234567890

Тип: group
Название: Мой проектный чат
```

### 2. Добавление чата для уведомлений

#### Через админ-панель:

1. Откройте админ-панель: `/admin`
2. Нажмите на кнопку **📱 Чаты**
3. Следуйте инструкциям:
   - Для **приватного чата**: просто отправьте /getChatId здесь
   - Для **группового чата**:
     - Добавьте бота в чат
     - Отправьте /getChatId в чат
     - Скопируйте ID и отправьте его в админ-панель
4. Отправьте ID чата сообщением в чат с админ-панелью

Пример:
```
Вы нажимаете "📱 Чаты"
Админ-панель: "Отправьте ID сообщением"
Вы: "-1001234567890"
Админ-панель: "✅ Чат -1001234567890 добавлен"
```

### 3. Удаление чата

1. Откройте админ-панель: `/admin`
2. Нажмите на кнопку **📱 Чаты**
3. Нажмите на кнопку **❌** рядом с чатом, который хотите удалить
4. Чат будет удален из списка

## Типы чатов

### Приватные чаты (personal)
- **ID**: положительное число (например, `123456789`)
- **Как получить**: отправить `/getChatId` в личном чате с ботом
- **Примечание**: бот общается с вами напрямую

### Групповые чаты
- **ID**: отрицательное число (например, `-1001234567890`)
- **Как получить**:
  1. Добавить бота в групповой чат
  2. Отправить `/getChatId` в группе
  3. Бот ответит в личном чате или группе
- **Примечание**: бот должен быть участником группы

### Каналы
- **ID**: отрицательное число, начинающееся с `-100`
- **Как получить**: отправить `/getChatId` в канале
- **Примечание**: требуется административный доступ в канале

## Поведение

### Если нет добавленных чатов
- Уведомления отправляются в **основной чат** (из переменной `TELEGRAM_CHAT_ID` в `.env`)
- Это поведение по умолчанию для обратной совместимости

### Если добавлены чаты в админ-панели
- Уведомления отправляются во **все добавленные чаты**
- Основной чат из конфига игнорируется

### При ошибке отправки
- Бот продолжает отправку в остальные чаты
- Ошибка логируется в файл логов
- Парсер не прерывается

## Примеры

### Пример 1: Отправка уведомлений в групповой чат

1. Создаете групповой чат "Проекты витрины"
2. Добавляете бота в чат
3. Отправляете `/getChatId`
4. Бот ответит: "ID: -1001234567890"
5. Копируете ID
6. Открываете админ-панель (`/admin`)
7. Нажимаете "📱 Чаты"
8. Отправляете ID: `-1001234567890`
9. Теперь уведомления будут приходить в этот групповой чат

### Пример 2: Отправка уведомлений в несколько чатов

1. Добавляете приватный чат ID: `123456789`
2. Добавляете групповой чат ID: `-1001234567890`
3. Добавляете канал ID: `-1001987654321`
4. При новом проекте уведомление отправляется в ВСЕ ТРИ чата одновременно

## Технические детали

### База данных

Таблица `notification_chats`:
- `id` - уникальный ID записи
- `chat_id` - ID чата в Telegram (основной ключ)
- `chat_name` - опциональное название для удобства
- `is_active` - флаг активности (по умолчанию 1)
- `created_at` - время добавления

### API методы

```python
# Repository методы
repo.add_notification_chat(chat_id: str, chat_name: Optional[str] = None)
repo.remove_notification_chat(chat_id: str)
repo.get_notification_chats() -> List[NotificationChat]  # только активные
repo.get_all_notification_chats() -> List[NotificationChat]  # все

# TelegramService методы
await telegram.send_notification(message: str, chat_ids: Optional[List[str]] = None)
await telegram.send_alert(error_message: str, chat_ids: Optional[List[str]] = None)
```

## Решение проблем

### Бот не отправляет уведомления в групповой чат

**Проверьте:**
1. Бот добавлен в групповой чат?
2. У бота есть права на отправку сообщений?
3. ID чата правильный (должен быть отрицательным)?

**Решение:**
- Удалите чат из админ-панели
- Переасигнете ID командой `/getChatId` в группе
- Добавьте ID заново

### Ошибка "Chat not found"

**Причины:**
- ID неправильный
- Бот был удален из чата/канала
- Чат был заблокирован или удален

**Решение:**
- Удалите этот чат из админ-панели
- Проверьте, что бот все еще участник чата
- Добавьте чат заново

### Уведомления идут только в один чат

**Вероятная причина:**
- В админ-панели не добавлено много чатов

**Решение:**
- Откройте админ-панель и проверьте список чатов в "📱 Чаты"
- Добавьте нужные чаты

## Рекомендации

1. **Безопасность**: Не делитесь ID группах с посторонними
2. **Производительность**: Отправка в 3-4 чата не влияет на скорость
3. **Мониторинг**: Проверяйте логи на ошибки отправки: `tail -f logs/parser-*.log`
4. **Резервная копия**: Основной чат из `.env` все еще работает как fallback
